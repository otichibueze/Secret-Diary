Description

Everyone makes mistakes. This situation could also happen with users. Imagine that they mistype a word in the input text and save it. After that, they realize the typo in the diary and do not want to leave it as it is. Users should be able to remove the last note from diary, and this is what you need to implement in this stage using an "Undo" button.

But what if users accidentally taps undo button after writing a really long text? To avoid such misclicks, an AlertDialog should appear asking users if they are sure about deleting the last note.

Objectives
  1. Create a new Button with id attribute btnUndo and text set to "Undo";
  2. On btnUndo click, an AlertDialog should be displayed with the following properties:
    • Title: "Remove last note"
    • Message: "Do you really want to remove the last writing? This operation cannot be undone!"
    • Positive Button: "Yes"
    • Negative Button: "No"
    • When importing AlertDialog make sure to import android.app.AlertDialog
  3. If users selects Yes, then remove the last writing from diary, otherwise, nothing should happen;
  4. Handle exceptional usage: If users selects Yes on AlertDialog, but the diary is empty, then nothing should happen;

Obs: Do not use mutation in global variables because tests do not support that.
