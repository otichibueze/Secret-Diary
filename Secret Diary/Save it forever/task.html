Description

We have created a nice diary so far. Users can add as many notes as they want, and remove the last one. But the application should also be able to persistently save data, and that is what you have to implement in this stage. When a text is added or removed all diary text should be saved including the date and time notations. When the user launches the application, they should see the diary exactly as before it was closed.

Objectives

  1. Save the diary persistently into SharedPreferences with name PREF_DIARY and key KEY_DIARY_TEXT. The diary should be saved after each Save and Undo action.

  2. The content to be saved is the text on textView as a single string with the same structure, that is, last entry saved is first entry displayed with entries separated with a blank line

  3. When users launches the app, the saved diary should be restored.

  4. The two buttons should work, like in the previous stage, also for the notes loaded from persistent storage.

    Obs: Do not use mutation in global variables because tests do not support that.
