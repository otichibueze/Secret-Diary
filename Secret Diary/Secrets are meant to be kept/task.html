Description

The diary is ready to use; well done! Since our diary keeps our secrets tight, we don't want others to access and read it. The last step is to protect the diary with a password known only to the user.

For this, create a new Activity that will handle the login. If the user correctly enters the secret number and taps the Log In button, then they can access the diary and work with it like in the previous stage. If the secret number is incorrect then let users know by showing them a error message.

Objectives

  1. Create a new Activity named LoginActivity with the following components and set it to be the launcher activity:

    • A TextView with id of tvTitle and text Secret Diary;

    • An EditText with id attribute set to etPin that only accepts the numeric input. Also, a hint Enter PIN should inform the user about the role of the input field;

    • A Button with btnLogin as id and Log In as text property;

  2. If the user presses the Log In button, and the entered PIN is correct (the default PIN used in tests: 1234), launch the diary Activity you implemented in the previous stages. Otherwise, set the error property of the input text field to Wrong PIN!

  3. The LoginActivity should not remain on backstack, which implies that pressing the back button on MainActivity should close the application instead of going back to LoginActivity

Obs: Tests assume the existence of LoginActivity class. Build will fail if that class has not been created yet. Do not use mutation in global variables because tests do not support that.
